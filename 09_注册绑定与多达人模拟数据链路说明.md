# 注册绑定与多达人模拟数据链路说明

## 当前实现

新用户注册时，系统会创建三类核心数据：

- `users`：登录账号。
- `creators`：达人数据空间，登录账号通过 `creator_id` 绑定到这里。
- `platform_accounts`：该达人模拟绑定的平台，例如抖音、B站、小红书。

注册成功后，后端会按用户选择的平台复制一份初始化数据，写入：

- `platform_accounts`
- `videos`
- `video_metric_snapshots`
- `video_traffic_source_metrics`
- `creator_metric_snapshots`
- `audience_profile_snapshots`
- `insights`
- Spark 结果表的初始聚合数据

这样新注册账号第一次进入页面时不会是空白页。

## 运行链路

所有达人共用一条数据管道：

```text
mock_generator
-> Flume spooldir
-> Kafka video_stats_topic
-> Spark Streaming
-> MySQL
-> Flask API
-> Vue 前端
```

数据隔离依赖事件里的 `creator_id`。前端登录后只访问 `/api/me/...`，后端从 session 找到当前用户的 `creator_id`，再读取对应数据。

## Mock 生成器变化

`mock_generator/generate_events.py` 已支持数据库驱动模式：

```bash
python3 mock_generator/generate_events.py \
  --batch-size 12 \
  --interval 30 \
  --env-file /opt/creatorpulse/app/.env.generator \
  --require-db
```

`.env.generator` 示例：

```env
MYSQL_HOST=192.168.88.1
MYSQL_PORT=3306
MYSQL_DATABASE=creatorpulse
MYSQL_USER=spark_user
MYSQL_PASSWORD=spark_password
```

`--require-db` 表示必须从 MySQL 读取绑定平台和视频模板。如果数据库配置缺失、PyMySQL 未安装、或没有绑定平台数据，脚本会直接失败，不会偷偷回退到旧的固定 mock 数据。

## 后续建议

下一步可以把下面三个进程做成固定启动脚本：

- Flume 常驻。
- `mock_generator --require-db` 常驻。
- Spark Streaming 常驻。

这样注册新用户后，生成器下一轮读取 MySQL 时会自动发现新 `creator_id` 和绑定平台，并开始给这个用户生成持续事件。
