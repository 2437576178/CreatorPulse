## 1. 项目概述

### 1.1 项目名称

**CreatorPulse — 达人粉丝增长分析与智能创作建议平台**

### 1.2 项目简介

2025-2026 年，内容达人通常同时在抖音、B站、小红书、视频号、YouTube、微博等多个平台发布内容。各平台后台只能展示各自的数据，达人很难统一判断：哪些视频真正带来了粉丝增长，哪些互动代表粉丝粘性，哪些平台和内容类型值得继续投入。

CreatorPulse 通过模拟多平台开放 API 实时采集播放、点赞、评论、分享、收藏、粉丝变化、弹幕、话题热度等数据，利用 Kafka + Spark Streaming 进行实时聚合，SparkSQL 进行离线复盘，Spark MLlib 输出选题、发布时间、标题封面、平台侧重等智能创作建议。Flask + Vue 提供达人个人增长分析网站。

### 1.3 核心增长链路

```text
曝光播放 → 互动行为 → 粉丝转化 → 粉丝粘性 → 内容复盘 → 智能创作建议
```

本项目不以 MCN 多角色管理后台为主线，而是聚焦达人本人如何理解自己的内容表现，并指导下一次创作。

### 1.4 核心价值主张

| 维度 | 价值 |
|------|------|
| 🔄 **跨平台聚合** | 一个工作台查看 6 个平台的播放、互动、粉丝和内容分布 |
| 📈 **粉丝增长分析** | 判断账号粉丝增长是在加速、放缓还是异常波动 |
| 🎯 **粉丝转化分析** | 找出哪些视频把播放和互动真正转化成了关注 |
| ❤️ **粉丝粘性分析** | 通过评论、收藏、分享、弹幕等行为判断粉丝质量 |
| 🔥 **机会发现** | 发现适合达人领域和粉丝画像的热点机会 |
| 🤖 **智能创作建议** | 输出选题、发布时间、标题封面、平台侧重建议 |

## 2. 业务背景与目标

### 2.1 业务背景

达人面临的核心痛点：

- **平台数据分散**：多个平台后台割裂，无法统一查看整体增长状态。
- **播放量解释不了增长**：视频播放高不代表涨粉，达人需要知道播放是否转化为关注。
- **不知道哪些视频真正转粉**：缺少单视频涨粉贡献和播放转粉率分析。
- **粉丝粘性难判断**：点赞、评论、收藏、分享、弹幕等行为分散在不同平台，难以判断粉丝是否持续互动。
- **内容分布不清晰**：不知道平台投入、内容类型和发布时间是否合理。
- **创作建议缺少依据**：下一条内容发什么、什么时候发、重点发哪个平台仍然依赖经验。

### 2.2 业务目标

| 目标 | 衡量指标 | 目标值 |
|------|----------|--------|
| 数据聚合延迟 | 各平台数据到统一视图 | < 10 秒 |
| 降低查看成本 | 日常跨平台数据查看时间 | 降低 80% |
| 提升粉丝转化 | 播放转粉率 / 互动转粉率 | 持续提升 |
| 提升粉丝净增长 | 今日净增粉丝、7 日净增趋势 | 持续提升 |
| 提升粉丝粘性 | 评论率、收藏率、分享率、弹幕密度 | 持续提升 |
| 解释推荐结果 | 创作建议给出依据、预计收益和关联数据 | 100% 建议可解释 |

## 4. 核心指标体系

| 指标组 | 核心指标 | 说明 |
|------|------|------|
| **账号增长健康度** | 粉丝增长趋势、转粉率、粉丝粘性、内容效率 | 综合判断账号整体增长状态 |
| **粉丝增长** | 总粉丝、新增粉丝、掉粉数、净增粉丝、增长率 | 判断账号是否持续增长 |
| **粉丝转化** | 播放转粉率、互动转粉率、主页访问转粉率、单视频涨粉贡献 | 判断内容是否能把流量变成粉丝 |
| **粉丝粘性** | 评论率、收藏率、分享率、弹幕密度、连续互动行为 | 判断粉丝是否愿意持续互动 |
| **视频表现** | 播放量、点赞、评论、分享、收藏、完播率、平均观看时长 | 分析单条视频表现 |
| **视频分布** | 平台分布、内容类型分布、发布时间分布、流量来源分布 | 判断内容结构和平台投入是否合理 |
| **机会建议** | 话题适配度、选题建议、发布时间建议、标题封面建议、平台侧重建议 | 指导下一次创作 |

### 4.1 指标口径

| 指标 | 计算口径 |
|------|----------|
| 播放转粉率 | 新增粉丝数 / 播放量 |
| 互动转粉率 | 新增粉丝数 / 互动总数 |
| 互动总数 | 点赞数 + 评论数 + 分享数 + 收藏数 + 弹幕数 |
| 粉丝净增长 | 新增粉丝数 - 掉粉数 |
| 单视频涨粉贡献 | 视频发布后统计窗口内带来的新增粉丝数 |
| 内容效率 | 单位内容带来的播放、互动和涨粉综合表现 |
| 账号增长健康度 | 粉丝增长趋势 30% + 转粉率 25% + 粉丝粘性 25% + 内容效率 20% |
| 话题适配度 | 话题热度、达人历史内容类型、粉丝画像、平台表现的综合评分 |

| **账号增长健康度** | 综合展示粉丝增长、转粉率、粉丝粘性、内容效率 | 10s |
| **粉丝增长趋势** | 总粉丝、今日新增、今日掉粉、净增粉丝趋势 | 10s |
| **粉丝转化漏斗** | 播放 → 互动 → 主页访问 → 关注 | 10s |
| **粉丝粘性信号** | 评论率、收藏率、分享率、弹幕密度 | 30s |
| **最新视频涨粉表现** | 最近发布视频的播放增长、涨粉曲线和转粉率 | 5s |
| **视频分布情况** | 各平台、内容类型、发布时间的视频占比和效率 | 30s |
| **热点机会摘要** | 适合达人跟进的话题、热度和适配度 | 60s |
| **下一步创作建议** | 选题、发布时间、标题封面、平台侧重摘要 | 60s |

### 5.2 粉丝分析

| 功能 | 描述 |
|------|------|
| 粉丝增长趋势 | 按平台和时间查看粉丝新增、掉粉、净增变化 |

| 粉丝画像 | 查看粉丝年龄、性别、地域、活跃时间段 |

### 5.3 视频分析

| 功能 | 描述 |
|------|------|
| 视频列表 | 展示视频封面、标题、平台、播放量、互动率、涨粉数、播放转粉率 |
| 单视频详情 | 查看播放增长曲线、涨粉曲线、互动趋势、生命周期 |

| 评论/弹幕反馈 | 分析评论情感、弹幕密度和负面评论变化 |

### 5.4 内容分布

| 功能 | 描述 |
|------|------|
| 平台分布 | 分析各平台视频数、播放占比、粉丝占比、涨粉效率 |
| 内容类型分布 | 分析不同内容类型的播放、互动、转粉和粘性表现 |
| 发布时间分布 | 按星期和小时分析播放量、互动率、转粉率 |
| 流量来源分布 | 分析推荐流、搜索、关注、分享等来源占比 |
| 平台投入产出 | 对比平台内容投入占比与涨粉贡献占比 |

### 5.5 机会建议

机会建议页面包含 3 个 Tab：

| Tab | 功能 | 描述 |
|------|------|------|
| 热点机会 | 话题雷达、话题适配度、风险提示 | 发现适合达人跟进的话题 |
| 创作建议 | 选题、发布时间、标题封面、平台侧重 | 输出下一条内容的执行建议 |
| 参考洞察 | 同领域参考账号内容特征 | 分析高转粉内容特征，不作为独立竞品管理 |

每条建议必须包含推荐结论、推荐理由、依据数据、预计收益和关联视频/话题。

### 5.6 个人中心

## 6. 数据分析需求

### 6.1 实时分析（Spark Streaming）

| 序号 | 分析指标 | 计算方式 | 窗口 | 输出 |
|------|----------|----------|------|------|
| RT-01 | 实时总播放量 | 各平台播放量求和 | 滑动 10s/5s | 增长总览 |
| RT-02 | 各平台播放量分布 | 按 platform 分组求和 | 滑动 30s/10s | 内容分布 |
| RT-03 | 实时互动率 | 互动总数 / 播放量 | 滑动 30s/10s | 粉丝粘性 |
| RT-04 | 粉丝实时增长 | 粉丝变化事件求和 | 滑动 30s/10s | 粉丝分析 |
| RT-05 | 实时播放转粉率 | 新增粉丝数 / 播放量 | 滑动 30s/10s | 粉丝转化 |
| RT-06 | 实时互动转粉率 | 新增粉丝数 / 互动总数 | 滑动 30s/10s | 粉丝转化 |
| RT-07 | 新视频涨粉表现 | 按 video_id 统计播放增长、涨粉数、转粉率 | 滑动 30s/10s | 视频分析 |
| RT-08 | 热点话题检测 | 多平台标签频次突增检测 | 滑动 120s/60s | 热点机会 |
| RT-09 | 异常流量与掉粉检测 | 播放量异常飙升/暴跌、掉粉异常 | 滑动 60s/30s | 通知提醒 |
| RT-10 | 实时弹幕密度 | 弹幕数量 / 分钟 | 滑动 30s/10s | 粉丝粘性 |

### 6.3 智能推荐（Spark MLlib）

| 序号 | 推荐策略 | 算法 | 输入 | 频率 |
|------|----------|------|------|------|
| RC-01 | 选题推荐 | 协同过滤 + 时间序列预测 | 历史视频表现 + 内容类型趋势 + 转粉率 | 每天 |
| RC-02 | 发布时间推荐 | K-Means 聚类 + 回归 | 粉丝活跃时段 + 历史发布时间效果 | 每天 |
| RC-03 | 标题优化建议 | TF-IDF + 回归分析 | 标题关键词 vs 转粉率相关性 | 每周 |
| RC-04 | 封面优化建议 | 特征评分模型 | 封面特征 + 点击/收藏/转粉表现 | 每周 |
| RC-05 | 热点机会适配 | 时序异常检测 + 适配度评分 | 话题趋势 + 粉丝画像 + 历史内容类型 | 实时 |
| RC-06 | 平台侧重建议 | 多维度评分模型 | 各平台互动率、转粉率、涨粉效率、粘性 | 每周 |
| RC-07 | 参考账号洞察 | 关联规则分析 | 参考账号内容特征 + 高转粉表现 | 每天 |

## 9. 系统约束与假设

- ❌ 不使用 Redis、Docker

- ✅ Kafka 地址：node1:9092
