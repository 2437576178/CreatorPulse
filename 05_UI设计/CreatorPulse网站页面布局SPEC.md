## 1. 产品定位

CreatorPulse 定位为**达人粉丝增长分析与智能创作建议平台**，不是 MCN 多角色管理后台，也不是管理员运维平台。

网站服务对象是“达人本人”：一个同时运营抖音、B站、小红书、视频号、YouTube、微博等平台的内容创作者。页面设计要帮助达人回答这些问题：

- 我的粉丝从哪里来？
- 哪些视频最能带来粉丝转化？
- 粉丝增长是在加速还是放缓？
- 粉丝是否有粘性，是否持续互动、收藏、评论、回访？
- 我的内容类型、平台分布、发布时间是否合理？
- 下一条内容应该发什么、什么时候发、重点发哪个平台？

因此，后续网站不是围绕“角色权限”展开，而是围绕**达人增长链路**展开：

```text
曝光播放 → 互动行为 → 粉丝转化 → 粉丝粘性 → 内容复盘 → 智能创作建议
```

`creatorpulse-demo4.html` 仍作为“增长总览大屏”的视觉基准，但它只是完整达人工作台中的首页。

## 2. 核心指标体系

页面应优先展示达人真正关心的增长指标，而不是泛泛的播放量大屏。

| 指标组 | 核心指标 | 说明 |
|---|---|---|
| 粉丝增长 | 总粉丝、新增粉丝、净增粉丝、掉粉数、增长率 | 判断账号是否持续增长 |
| 粉丝转化 | 播放转粉率、互动转粉率、单视频涨粉贡献 | 判断内容是否能把流量变成粉丝 |
| 粉丝粘性 | 互动率、评论率、收藏率、复访/持续互动、弹幕密度 | 判断粉丝是否只是路过，还是愿意持续互动 |
| 视频分布 | 平台分布、内容类型分布、发布时间分布、视频生命周期 | 判断内容结构和平台投入是否合理 |
| 内容表现 | 播放、点赞、评论、分享、收藏、完播率、平均观看时长 | 分析单条视频表现 |
| 创作建议 | 选题建议、发布时间建议、标题/封面建议、平台侧重建议 | 直接服务下一次发布 |

## 3. 导航模型

### 3.1 导航原则

导航应以达人个人工作流为中心，而不是以系统角色为中心。

左侧主导航建议只保留达人高频入口：

| 图标语义 | 页面名称 | 路由建议 | 页面目标 |
|---|---|---|---|
| 首页 | 增长总览 | `/dashboard/growth` | 一屏看清账号整体增长状态 |
| 用户/粉丝 | 粉丝分析 | `/fans` | 看粉丝增长、转化、粘性和画像 |
| 视频 | 视频分析 | `/videos` | 看视频表现、涨粉贡献和内容结构 |
| 分布/图表 | 内容分布 | `/distribution` | 看平台、类型、发布时间、流量来源分布 |
| 火焰 | 机会建议 | `/opportunities` | 发现适合跟进的话题，并生成选题、发布时间、标题封面、平台侧重建议 |
| 设置 | 个人中心 | `/profile` | 查看个人报告，绑定平台账号，设置采集频率和通知偏好 |

竞品追踪不作为一级导航。它可以作为“机会建议”里的辅助模块，改名为“参考账号洞察”，只用于帮助达人改进内容。

### 3.2 顶部 Tab 规则

顶部 pill 是当前页面内的二级视图。

示例：

- 增长总览：`实时概览`、`粉丝转化`、`粉丝粘性`、`视频分布`
- 粉丝分析：`增长趋势`、`转化来源`、`粘性行为`、`粉丝画像`
- 视频分析：`最新视频`、`涨粉贡献`、`互动质量`、`生命周期`
- 内容分布：`平台分布`、`内容类型`、`发布时间`、`流量来源`
- 机会建议：`热点机会`、`创作建议`、`参考洞察`
- 个人中心：`数据报告`、`平台绑定`、`采集设置`、`通知偏好`

## 5. 页面布局规格

### 5.1 全局 App Shell

登录后页面共享同一套外壳：

- 背景：创作工作台、多屏数据监控、视频剪辑等真实创作场景图。
- 主容器：沿用 `creatorpulse-demo4.html` 的玻璃拟态大面板。
- 左侧：达人工作流主导航。
- 顶部：品牌、当前页面 Tab、数据同步状态、通知、达人头像。
- 主内容：根据页面分为大屏型、分析型、列表型和设置型。

全局状态区建议持续显示：

- 当前同步状态：例如 `5s Sync`。
- 已绑定平台数：例如 `6 platforms`。
- 今日新增粉丝：例如 `+3.2万`。
- 通知入口：热点、异常流量、负面评论突增。

### 5.2 增长总览页 `/dashboard/growth`

视觉基准：`creatorpulse-demo4.html`。

页面职责：

- 一屏看清达人当前增长状态。
- 重点不是“播放量炫技”，而是“播放是否转成粉丝、粉丝是否有粘性”。

建议模块：

| 区域 | 模块 | 展示内容 | 点击跳转 |
|---|---|---|---|
| 左侧大圆环 | 账号增长健康度 | 粉丝增长趋势、转粉率、粉丝粘性、内容效率综合评分 | `/fans?tab=growth` |
| 左下指标组 | 粉丝转化、粉丝粘性、爆发速度 | 播放转粉率、互动率、收藏率、5s 增长 | 对应分析页 |
| 顶部 6 节点 | 平台、内容、总播放、新增粉丝、转粉率、同步延迟 | 核心状态摘要 | 数据详情 |
| 白色大卡 | 粉丝转化面板 | 播放、互动、新粉之间的转化关系 | `/fans?tab=conversion` |
| 紫色大卡 | 最新视频涨粉表现 | 最近视频发布后涨粉曲线 | `/videos?sort=latest` |
| 底部卡 1 | 视频分布情况 | 各平台/内容类型的视频占比 | `/distribution` |
| 底部卡 2 | 粉丝粘性信号 | 评论、收藏、分享、弹幕、复访趋势 | `/fans?tab=stickiness` |
| 底部卡 3 | 下一步创作建议 | 推荐发布时间、选题方向、重点平台 | `/opportunities?tab=recommendations` |

需要把当前 `creatorpulse-demo4.html` 的主圆环从“全平台总播放”调整为“账号增长健康度”。播放量仍可展示，但不能是唯一主角。

### 5.3 粉丝分析页 `/fans`

页面职责：

- 回答“粉丝为什么涨、从哪里涨、涨来的粉丝有没有价值”。

推荐布局：

- 顶部 KPI：总粉丝、今日新增、今日掉粉、净增粉丝、粉丝增长率、播放转粉率。
- 主图：粉丝增长趋势折线，支持按平台切换。
- 转化漏斗：播放 → 互动 → 主页访问 → 关注。
- 粘性行为卡片：评论率、收藏率、分享率、弹幕密度、连续互动粉丝占比。
- 粉丝画像：年龄、性别、地域、活跃时间段。

顶部 Tab：

- `增长趋势`
- `转化来源`
- `粘性行为`
- `粉丝画像`

### 5.4 视频分析页 `/videos`

页面职责：

- 回答“哪些视频带来增长，哪些视频只有播放没有转粉”。

推荐布局：

- 顶部筛选：平台、内容类型、发布时间、增长状态。
- 视频列表：封面、标题、平台、播放量、互动率、涨粉数、播放转粉率、完播率。
- 排行卡片：
  - 涨粉贡献 TOP 5。
  - 高播放低转粉视频。
  - 高粘性视频。
  - 正在加速的视频。
- 单视频详情：
  - 播放增长曲线。
  - 涨粉曲线。
  - 互动趋势。
  - 评论情感。
  - 弹幕密度。
  - 推荐复盘：标题、封面、发布时间、内容类型是否有效。

### 5.5 内容分布页 `/distribution`

页面职责：

- 回答“我把内容发在哪里、发什么类型、什么时候发，是否合理”。

推荐布局：

- 平台分布：各平台视频数、播放占比、粉丝占比、涨粉效率。
- 内容类型分布：教程、测评、种草、Vlog、直播切片等类型表现。
- 发布时间分布：星期 × 小时热力图，对比播放量、互动率、转粉率。
- 流量来源分布：推荐流、搜索、关注、分享、其他。

关键指标：

- 平台投入产出比：该平台视频占比 vs 该平台涨粉占比。
- 内容类型效率：该类型播放占比 vs 转粉占比。
- 时间段效率：发布时间段 vs 粉丝活跃时段。

### 5.6 机会建议页 `/opportunities`

页面职责：

- 把“热点机会”和“创作建议”合并为一个达人行动页。
- 不是泛热点榜，而是“适合这个达人跟进的机会”，并直接给出下一步创作建议。

页面固定拆为 3 个 Tab：

| Tab | 内容 | 布局 |
|---|---|---|
| 热点机会 | 话题雷达、话题适配度、风险提示 | 大面积热力图/气泡矩阵 + 话题卡片 |
| 创作建议 | 选题、发布时间、标题封面、平台侧重 | 推荐卡片 + 依据图表 + 执行清单 |
| 参考洞察 | 同领域账号高转粉内容特征 | 对比表格 + 内容特征提取 |

热点机会 Tab：

- 话题雷达：跨平台热度、增长率、平台覆盖数。
- 适配度评分：结合达人历史内容类型、粉丝画像、平台表现。
- 风险提示：负面评论、舆情风险、热点衰退。
- 关联视频：达人过去是否做过类似内容，效果如何。

点击行为：

- 点击话题进入详情抽屉。
- “生成选题”在当前页切到 `创作建议` Tab。
- “查看相关视频”跳转到视频分析页。

创作建议 Tab：

- 左侧：推荐卡片列表。
- 中间：推荐依据图表。
- 右侧：执行清单。

推荐类型：

| 推荐类型 | 示例 |
|---|---|
| 选题推荐 | “近期通勤妆教程转粉率高于均值 35%，建议继续做同类内容。” |
| 发布时间推荐 | “你的粉丝周五 20:00-21:00 最活跃，预计播放转粉率最高。” |
| 标题优化 | “标题包含‘新手必看’的视频平均转粉率高 42%。” |
| 封面优化 | “高饱和度封面在小红书收藏率更高。” |
| 平台侧重 | “B站互动粘性最高，适合发布深度教程；抖音适合短爆款引流。” |

每条推荐必须包含：

- 推荐结论。
- 推荐理由。
- 依据数据。
- 预计收益。
- 关联视频或话题。

参考洞察 Tab：

- 展示同领域参考账号近期高转粉视频特征。
- 不做独立竞品管理，不展示组织级对标报表。
- 只输出可借鉴的选题、标题、发布时间和内容结构特征。

### 5.7 个人中心页 `/profile`

页面职责：

- 合并“数据报告”和“账号设置”，作为低频功能入口。
- 给达人做阶段性复盘，并管理平台绑定、采集频率和通知偏好。

页面固定拆为 4 个 Tab：

| Tab | 内容 | 布局 |
|---|---|---|
| 数据报告 | 个人日报、周报、月报 | 报告卡片 + 摘要指标 + 导出入口 |
| 平台绑定 | 各平台授权状态 | 平台列表 + 授权状态 |
| 采集设置 | 采集频率和同步策略 | 表单配置 |
| 通知偏好 | 热点、异常、掉粉、负面评论提醒 | 开关列表 |

- 顶部：达人资料、绑定平台数、当前同步状态。
- 左侧 Tab：数据报告、平台绑定、采集设置、通知偏好。
- 主区域根据 Tab 切换内容。

数据报告：

- 个人日报：今日播放、互动、涨粉、最佳视频、异常提醒。
- 个人周报：内容类型表现、粉丝增长来源、转粉率变化。
- 个人月报：账号成长趋势、平台投入产出、创作建议总结。

不建议把报告中心做成老板经营报表，应保持个人创作者语气。

账号设置：

| 模块 | 内容 |
|---|---|
| 平台账号绑定 | 抖音、B站、小红书、视频号、YouTube、微博授权状态 |
| 数据采集频率 | 默认 5 秒、30 秒、60 秒等采集策略 |
| 通知偏好 | 热点提醒、异常流量提醒、负面评论提醒 |
| 个人资料 | 昵称、头像、领域标签 |

不展示复杂 RBAC、MCN 管理员、超管权限、Topic 运维状态。Topic 状态可以作为课程技术说明存在，但不进入达人产品主导航。

## 8. 视觉与交互原则

1. 首页是达人增长总览，不是泛数据驾驶舱。
2. 播放量不能独占主视觉，应与粉丝增长、转粉率、粘性指标一起呈现。
3. 页面语言使用“你的账号”“你的粉丝”“你的内容”，强化个人创作者视角。
4. 参考账号、热点、推荐都服务于达人增长，不做独立 MCN 管理后台。
5. 数据中心类页面要可筛选、可对比，但不要堆过多企业运营指标。
6. 个人中心只保留达人能理解和会使用的报告与配置。
7. 所有推荐必须给出依据和预计收益，避免空泛建议。
8. 保留 `demo4.html` 的玻璃拟态视觉，但分析页要比首页更清晰、可读。

## 9. 后续实现建议

建议按以下顺序实现静态原型：

1. `growth-dashboard.html`：达人增长总览，基于 `creatorpulse-demo4.html` 改造。
2. `fans-analysis.html`：粉丝增长、转化、粘性、画像。
3. `video-analysis.html`：视频列表、涨粉贡献、单视频详情。
4. `content-distribution.html`：平台、类型、发布时间、流量来源分布。
5. `creator-opportunities.html`：热点机会、选题建议、发布时间、标题封面、平台侧重建议。
6. `creator-profile.html`：个人日报、周报、月报、平台绑定、采集频率、通知偏好。

## 10. 实施检查点

实施和后续 UI 原型制作时需要检查：

1. “粉丝转化率、粉丝增长、粉丝粘性、视频分布”是否已经成为页面主线。
2. 是否还保留了过多 MCN/管理员视角，应继续删除。
3. 增长总览页的主圆环是否明确展示“账号增长健康度”。
4. 机会建议页是否清晰分成 `热点机会`、`创作建议`、`参考洞察` 3 个 Tab。
5. 个人中心是否清晰分成 `数据报告`、`平台绑定`、`采集设置`、`通知偏好` 4 个 Tab。
6. 所有推荐是否都包含推荐依据和预计收益。

---

## 11. 当前结论

CreatorPulse 的网站应以达人个人增长为中心。完整页面结构应围绕“粉丝增长、粉丝转化、粉丝粘性、视频表现、内容分布、创作建议”展开。`creatorpulse-demo4.html` 不是整站，而是“增长总览首页”的视觉基准；后续页面都应服务于达人理解自己的内容表现并指导下一次创作。
